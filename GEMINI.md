# Project Context: Technical Interview Preparation

This repository is a curated collection of organized notes and real-world interview questions for software engineering roles, optimized for display on GitHub Pages.

## 📂 Directory Overview

The project is structured by technical domain, with each directory containing a dedicated markdown file for that topic.

- **`ai/`**: Artificial Intelligence (MCP, Agentic AI, RAG, LLM fundamentals).
- **`aws/`**: Amazon Web Services (EC2, S3, Lambda, Cloud Computing).
- **`db/`**: Databases (Indexing, ACID, Sharding, NoSQL).
- **`design-pattern/`**: Software Design Patterns (MVC, structural patterns).
- **`hr/`**: HR Interview Preparation (Common questions, Soft Skills).
- **`js/`**: JavaScript Fundamentals (Closures, Hoisting, Promises, ES6+).
- **`node/`**: Node.js (Event Loop, Streams, Worker Threads, Security).
- **`react/`**: React (Virtual DOM, Hooks, Reconciliation, React 19).
- **`sql/`**: SQL (Complex Queries, Joins, Optimization).
- **`system-design/`**: System Design (Scalability, Load Balancing, Microservices).

## 🔑 Key Files

- **`index.md`**: The landing page for the GitHub Pages site, providing a high-level table of contents.
- **`README.md`**: Project overview and quick links to topics.
- **Topic Files (e.g., `js/js.md`)**: Detailed notes and interview questions for specific domains.

## 🛠️ Usage & Format

- **Format**: All notes use Markdown with Jekyll-style front matter for site navigation.
- **Callouts**: Look for `[!TIP]`, `[!NOTE]`, and `[!IMPORTANT]` blocks for high-impact "Interview Gold" tips.
- **GitHub Pages**: The project is designed to be hosted via GitHub Pages, likely using a theme compatible with Jekyll front matter.

## 🚀 Development Guidelines

- **Adding New Topics**: Create a new directory and a corresponding `.md` file. Ensure the front matter includes `layout: default`, a `title`, and a unique `nav_order`.
- **Content Style**: Maintain the "Interview Gold" callouts for key takeaways. Use clear headings and code examples where applicable.
- **Links**: Use relative links for navigation between documents.
